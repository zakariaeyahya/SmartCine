concept de base
🧠 Cours introductif – Web sémantique & Système de recommandation
1️⃣ Pourquoi le Web sémantique pour la recommandation ?
Problème classique
Les bases de données classiques :
* stockent des données
* mais ne comprennent pas leur sens
Exemple :
Film = Inception
Acteur = Leonardo DiCaprio


➡ Une base SQL ne sait pas que deux films sont liés par un acteur commun sans logique explicite.
Solution : Web sémantique
Le Web sémantique permet de :
* représenter les données avec du sens
* exprimer des relations explicites
* interroger les connaissances intelligemment
👉 C’est exactement ce dont on a besoin pour un système de recommandation.
________________


2️⃣ Concepts fondamentaux à connaître
🔹 2.1 RDF – Resource Description Framework
RDF est le format de base du Web sémantique.
Il représente les données sous forme de triplets :
Sujet — Prédicat — Objet


Exemple :
Inception — hasActor — LeonardoDiCaprio


👉 Tout dans le projet sera stocké sous cette forme.
________________


🔹 2.2 URI – Identifiants uniques
Chaque élément est identifié par une URI (adresse unique).
Exemple :
http://example.org/film#Inception


Pourquoi ?
* éviter les ambiguïtés
* permettre le partage et l’interopérabilité
________________


🔹 2.3 Namespace
Pour simplifier les URI, on utilise des namespaces.
Exemple :
ns = Namespace("http://example.org/film#")


Ensuite :
ns:Inception
ns:hasActor


👉 Utilisé partout : RDF, SPARQL, Protégé.
________________


3️⃣ OWL – Ontologies
🔹 3.1 Qu’est-ce qu’une ontologie ?
Une ontologie est :
une description formelle des concepts d’un domaine et de leurs relations
Dans notre projet :
* concepts : Film, Acteur, Genre
* relations : hasActor, hasGenre, directedBy
________________


🔹 3.2 Éléments d’une ontologie OWL
Élément
	Rôle
	Exemple
	Classe
	Concept
	Film
	Individu
	Instance réelle
	Inception
	Propriété objet
	Lien entre individus
	hasActor
	Propriété de données
	Valeur littérale
	releaseYear
	________________


4️⃣ Protégé – Éditeur d’ontologies
🔧 Qu’est-ce que Protégé ?
Protégé est :
* un éditeur graphique d’ontologies OWL
* utilisé pour créer :
   * classes
   * propriétés
   * individus
👉 Il sert à concevoir l’ontologie, pas à faire l’interface web.
________________


🔹 Rôle de Protégé dans le projet
Protégé permet de :
* modéliser le domaine cinéma
* visualiser les relations
* exporter l’ontologie en .owl ou .ttl
📦 Output :
films.owl / films.ttl


________________


5️⃣ Knowledge Graph
🔹 Définition
Un Knowledge Graph est :
* un ensemble de données RDF
* structuré par une ontologie
* stocké dans un triplestore
👉 Notre ontologie + individus = Knowledge Graph.
________________


6️⃣ Apache Jena Fuseki
🔧 Qu’est-ce que Fuseki ?
Apache Jena Fuseki est :
* un serveur SPARQL
* un triplestore RDF
Il permet :
* de stocker l’ontologie
* d’exécuter des requêtes SPARQL
* d’être interrogé via le web
________________


🔹 Rôle dans le projet
Fuseki est :
* la base de données sémantique
* le cœur du système de recommandation
URL typique :
http://localhost:3030/films


________________


7️⃣ SPARQL – Langage de requêtes
🔹 À quoi sert SPARQL ?
SPARQL est au Web sémantique ce que :
* SQL est aux bases relationnelles
Il permet de :
* rechercher des films
* trouver des relations
* générer des recommandations
________________


🔹 Types de requêtes utiles
Type
	Rôle
	SELECT
	Interroger
	INSERT DATA
	Ajouter des données
	CONSTRUCT
	Générer de nouvelles relations
	Exemple :
SELECT ?film WHERE {
  ?film :hasGenre :ScienceFiction .
}


________________


8️⃣ Logique de recommandation sémantique
Principe
On ne code pas la recommandation avec des if, mais avec :
* des requêtes SPARQL
* des relations sémantiques
Exemples de critères :
* même acteur
* même genre
* même réalisateur
👉 SPARQL = moteur de recommandation.
________________

2️⃣ Logique de recommandation
   * Définir les critères :
   * même acteur
   * même genre
   * même réalisateur
   * Écrire les requêtes SPARQL correspondantes
Exemple :
SELECT ?film WHERE {
  :Film_Inception :hasActor ?actor .
  ?film :hasActor ?actor .
  FILTER(?film != :Film_Inception)
}


👉 SPARQL joue le rôle du moteur de recommandation
________________

   * SPARQL

    :directedBy :ChristopherNolan ;
    :hasGenre :ScienceFiction ;
    :releaseYear 2010 ;
    :duration 148 .
}


✅ Option 4 — SPARQL Update AUTOMATIQUE (recommandé si CSV)
👉 Tu écris un script (Python ou Java) qui :

   * les envoie à Fuseki

Schéma :
CSV → Script → SPARQL INSERT → Fuseki


Exemple logique (pseudo-code) :
for film in csv:
    generate INSERT DATA {...}
    send to Fuseki
